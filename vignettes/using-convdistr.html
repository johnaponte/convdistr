Using the convdistr package

John J. Aponte

2018-10-31

The convdistr provide tools to define distribution objects and make mathematical operations with them. It keep track of the results as if they where scalar numbers but mantain the ability to obtain randoms samples of the convoluted distributions.

Practical example

What would be the resulting distribution of \(a+b*c\) if \(a\) is a normal distribution with mean 1 and standard deviation 0.5, \(b\) is a poisson distribution with lambda 5 and \(c\) is a beta distribution with shape parameters 10 and 20?

library(convdistr)
library(ggplot2)

a <- new_NORMAL(1,0.5)
b <- new_POISSON(5)
c <- new_BETA(10,20)
res <- a + b * c

metadata(res)
#>   distribution     rvar
#> 1  CONVOLUTION 2.666667
t(summary(res))
#>              [,1]         
#> distribution "CONVOLUTION"
#> varname      "rvar"       
#> oval         "2.666667"   
#> nsample      "10000"      
#> mean_        "2.677883"   
#> sd_          "1.013286"   
#> lci_         "0.9576428"  
#> median_      "2.568977"   
#> uci_         "4.956909"
ggDISTRIBUTION(res) + ggtitle("a + b * c")

The result is a distribution with expected value 2.67. A sample from 10000 drawns of the distribution shows a mean value of 2.68, a median of 2.57 and 95% centiles of 0.96, 4.96

The following sections describe the DISTRIBUTION object, how to create new DISTRIBUTION objects and how to make operations and mixtures with them.

The DISTRIBUTION object

The DISTRIBUTION is an abstract class (or interphase) that specific constructors should implement.

It contains 4 fields:

distribution : A character with the name of the distribution implemented

seed : A numerical seed that is use when obtaing random drawns for the summary function

oval : The observed value. It is the value expected. It is used as a number for the mathematical operations of the distributions as if they were a simple scalar

rfunc(n) : A function that generate random numbers from the distribution. Its only parameter n is the number of drawns of the distribution. It returns a matrix with as many rows as n, and as many columns as the dimensions of the distributions

The DISTRIBUTION object can support multidimensional distributions for example a dirchlet distribution. The names of the dimensions should coincides with the names of the oval vector. If it has only one dimension, the default name is rvar.

It is expected that the rfunc could be included in the creation of new distributions by convolution or mixture, so the environment should be carefully controlled to avoid reference leaking that is possible within the R language. For that reason, the rfunc should be created within a restrict_environment function that controls that only the variables that are required within the function are saved in the environment of the function.

Once the new objects are instanciated, the fields are immutable and should not be changed.

Methods

The following are methods for all objects of class DISTRIBUTION

  • metadata(x) Print the metadata for the distribution
  • summary(object, n=10000) Produce a summary of the distribution
  • rfunc(x, n) Generate n random drawns of the distribution
  • plot(x, n= 10000) Produce a density plot of the distribution
  • ggDISTRIBUTION(x, n= 10000) produce a density plot of the distribution using ggplot2
myDistr <- new_NORMAL(0,1)

metadata(myDistr)
#>   distribution rvar
#> 1       NORMAL    0

t(summary(myDistr))
#>              [,1]          
#> distribution "NORMAL"      
#> varname      "rvar"        
#> oval         "0"           
#> nsample      "10000"       
#> mean_        "-0.01430981" 
#> sd_          "1.010699"    
#> lci_         "-2.026211"   
#> median_      "-0.009265939"
#> uci_         "1.96156"

rfunc(myDistr, 10)
#>          rvar
#> 1  -1.7214367
#> 2   0.4592205
#> 3   1.0622453

Convolution

Mathematical operations between DISTRIBUTION like +, -, *, / can be perform with the new_CONVOLUTION(listdistr, op, omit_NA = FALSE) function. The listdistr paramater is a list of DISTRIBUTION objects on which the operation is made. A shorter version exists for each one of the operations as follow

  • new_SUM(listdistr, omit_NA = FALSE)
  • new_SUBSTRACTION(listdistr, omit_NA = FALSE)
  • new_MULTIPLICATION(listdistr, omit_NA = FALSE)
  • new_DIVISION(listdistr, omit_NA = FALSE)

but Mathematical operator can also be used.

d1 <- new_NORMAL(1,1)
d2 <- new_UNIFORM(2,8)
d3 <- new_POISSON(5)
dsum <- new_SUM(list(d1,d2,d3))
t(summary(dsum))
#>              [,1]         
#> distribution "CONVOLUTION"
#> varname      "rvar"       
#> oval         "11"         
#> nsample      "10000"      
#> mean_        "10.97604"   
#> sd_          "2.983288"   
#> lci_         "5.427262"   
#> median_      "10.90375"   
#> uci_         "17.03497"
t(summary(d1 + d2 + d3))
#>              [,1]         
#> distribution "CONVOLUTION"
#> varname      "rvar"       
#> oval         "11"         
#> nsample      "10000"      
#> mean_        "11.02101"   
#> sd_          "3.014313"   
#> lci_         "5.443241"   
#> median_      "10.91424"   
#> uci_         "17.14612"
ggDISTRIBUTION(dsum)

Mixture

A DISTRIBUTION, consisting on the mixture of several distribution can be obtained with the new_MIXTURE(listdistr, mixture) function where listdistr is a list of DISTRIBUTION objects and mixture the vector of probabilities for each distribution. If missing the mixture, the probability will be the same for each distribution.

d1 <- new_NORMAL(1,0.5)
d2 <- new_NORMAL(5,0.5)
d3 <- new_NORMAL(10,0.5)
dmix <- new_MIXTURE(list(d1,d2,d3))
t(summary(dmix))
#>              [,1]       
#> distribution "MIXTURE"  
#> varname      "rvar"     
#> oval         "5.333333" 
#> nsample      "10000"    
#> mean_        "5.326295" 
#> sd_          "3.72507"  
#> lci_         "0.2383337"
#> median_      "4.990554" 
#> uci_         "10.71557"
ggDISTRIBUTION(dmix)
